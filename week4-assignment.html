Week 4 Assignment: Terraform Infrastructure Deployment

⏱ ~60 minutes ✅ Free Tier 📚 Terraform + AWS Console

Overview

In this assignment, you will use Terraform to deploy a basic cloud infrastructure on AWS. This exercise reinforces the Infrastructure as Code (IaC) concepts covered in the Week 4 lecture and gives you hands-on experience with Terraform's core workflow: init, plan, apply, and destroy.

Objectives

  • Install and configure Terraform on your local machine
  • Write Terraform configuration to provision AWS resources
  • Deploy a VPC with public and private subnets, an EC2 instance, EBS volume, and S3 bucket
  • Practice safe teardown of infrastructure using terraform destroy

Prerequisites

  • An AWS account (free tier is sufficient)
  • AWS CLI configured with your credentials
  • Terraform installed (see Step 1 below)
  • A code editor (VS Code recommended)

Step-by-Step Instructions

1
Install Terraform (if you haven't already).
Follow the official installation guide for your operating system at developer.hashicorp.com/terraform/install. Verify your installation by running terraform --version in your terminal.
2
Set up a new project in Terraform.
Create a new directory for your project and initialize it with terraform init. You'll need at minimum a main.tf file with your AWS provider configuration and resource definitions.
3
(Optional but Recommended) Protect yourself from unintended charges.
Watch this video before setting anything up on AWS to avoid any unintended charges: Avoiding AWS Charges (YouTube).

For additional protection, set up a zero-spend budget: go to Billing and Cost Management from the menu on the top right (where your AWS account name is shown), then click Budgets in the left menu and Create budget. Select the zero-spend budget template.
Navigate to Billing and Cost Management from the top-right account menu
⚠ Important: The zero-spend budget will alert you if any charges begin to accrue. This is a safety net — all resources in this assignment are free-tier eligible when configured correctly.
4
Deploy a cloud infrastructure with the following components using Terraform:
  • Two Subnets – One named Public and one named Private. You can use mxtoolbox.com/subnetcalculator to figure out available IPs in the VPC.
  • One EC2 Instance – t2.micro deployed in the private subnet created above.
  • One EBS Volume – 8 GB. Attach it to the above EC2 instance.
  • One S3 Bucket – Ensure that the ACL is set to private to prevent any public access to your bucket.
5
For Windows Users:
Here is a tutorial that might be helpful as most docs on Terraform's website are based around Mac/Linux: Terraform on Windows Tutorial (YouTube).

Submission

6
Upload your Terraform code to your homework directory.
Just the main file will be fine. Please remove any access credentials (AWS access keys, secret keys) that you might have put in that file before uploading.
🚨 Security Reminder: Never commit AWS credentials (access keys, secret keys) to any repository or shared directory. Use environment variables or the AWS CLI credential file instead.
7
Upload a screenshot of your EC2 instance running in your AWS account.
Ensure that the screenshot includes your unique account name on the top right of the screen so we can verify it's your account.
📸 Completion Screenshot

Your screenshot should show:

  • The EC2 Instances page with your t2.micro instance in a Running state
  • Your AWS account name visible in the top-right corner of the console

Cleanup

8
Delete all your created Terraform resources.
Run terraform destroy in your project directory and confirm the destruction when prompted. This will remove all the resources you created and ensure you don't incur any ongoing charges.
🗑 Cleanup Checklist
  1. Run terraform destroy and type yes to confirm
  2. Verify in the AWS Console that all resources (EC2, EBS, S3, Subnets) have been removed
  3. Check your AWS Billing dashboard to confirm no unexpected charges
