AWS Cloud Networking — Virtual Private Cloud

What is a VPC?

A logically isolated section of the AWS Cloud spanning a whole region including multiple availability zones.

Default VPC & Subnet

Each AWS Account has a default VPC with a default Public Subnet in each AZ of the region.

Subnets

A range of IP Addresses in your VPC. Resides in a single AZ. Sort of like a partition on a hard drive.

Gateways

Horizontally Scaled, Redundant, Highly available VPC components that allow communication between VPC and the internet.

Security Groups

Controls traffic that is allowed to enter and leave the VPC. Associated with resources within a VPC.

Pricing

VPC by itself is free; its components such as NAT gateways, IP Addresses, etc. may cost.

AWS Databases
AWS DynamoDB

Advantages

  • NoSQL Database Service — Basically a bunch of JSON files
  • Supports document and key-value store models
  • Automatic data replication
  • Data backups to S3
  • Highly scalable
  • Only pay for what you use vs RDS where a set amount must be provisioned
  • Fast, very fast!

Limits

  • 64 KB limit on row size
  • Consistency is expensive
  • No relational models, no joins
  • Indexing can get expensive

AWS Compute — EC2
Amazon Elastic Compute Cloud (EC2) is a web service that provides resizable compute capacity in the cloud. In simpler terms, EC2 allows you to rent virtual servers (called instances) from Amazon's cloud infrastructure, which you can use to run your applications or services. You can choose the instance type based on your needs, whether it's for general-purpose computing, memory-intensive tasks, or high-performance computing.

AWS Compute — EC2 Pricing
On-Demand
  • Pay by the Hour
  • No long-term commitments
  • Most expensive
Saving Plans
  • Discounts of up to 72%
  • Committing use for 1-year / 3-year term
  • Steady availability
Spot
  • Save up to 90% by taking advantage of unused EC2 capacity
  • Only for fault tolerant or stateless workloads
  • Plan for interruptions and be flexible

AWS Compute — Lambda Pricing

Unit Conversions

Requests: 100/sec × 60 × 60 × 730 = 262,800,000 per month
Memory: 128 MB × 0.0009765625 = 0.125 GB
Ephemeral storage: 512 MB × 0.0009765625 = 0.5 GB

Compute Charges

262.8M requests × 50ms × 0.001 = 13,140,000 total compute seconds
0.125 GB × 13,140,000s = 1,642,500 GB-s
1,642,500 − 400,000 (free tier) = 1,242,500 billable GB-s
1,242,500 GB-s × $0.0000166667 = $20.71 USD / month

Request Charges

262,800,000 − 1,000,000 (free tier) = 261,800,000 billable requests
261,800,000 × $0.0000002 = $52.36 USD / month

Ephemeral Storage

0.5 GB − 0.5 GB (no additional charge) = $0.00 / month
Lambda Cost (with Free Tier): $20.71 + $52.36 = $73.07 USD / month

AWS Storage — S3
Can store any sort of files up to a maximum size of 5 TB.
Various storage classes help reduce overall costs:
  • General Purpose
  • Infrequent Access — millisecond retrieval
  • S3 Glacier Instant / Deep — Higher penalty for frequent access; deep can take up to 12 hours
  • Object storage
  • Pay as you Go — with options

AWS Storage — Elastic Block Store
  • Scale Fast
  • High Performance including replication within AZ
  • Optimizable Cost depending on performance needed
  • Allows encryption
  • Allows backups as EBS snapshots — Snapshot storage also costs
  • Can only be attached to one instance at a time

AWS Storage — Elastic File System
  • File level not block level
  • Unlimited scalability with automatic resizing as more storage is needed
  • Optimizable Cost depending on performance needed. Can scale higher than EBS
  • Has multi-AZ redundancy
  • Can be attached to multiple instances at the same time
