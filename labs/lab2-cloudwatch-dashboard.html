Lab 2: Building a CloudWatch Dashboard with a Billing Alarm

⏱ ~25 minutes ✅ Free Tier 📚 AWS Console Only

Overview

CloudWatch is AWS's monitoring and observability service. In this lab, you'll create an SNS notification topic, set up a billing alarm to detect unexpected charges, and build a CloudWatch dashboard to visualize your spending. This connects to the Prepare & Operate phases of Operational Excellence — specifically "Implement observability" and "Respond to events" from the Week 2 lecture.

Objectives

  • Create an SNS topic and email subscription for notifications
  • Set up a CloudWatch billing alarm to detect unexpected charges
  • Build a custom CloudWatch dashboard with billing widgets
  • Understand how monitoring and alerting support Operational Excellence

Prerequisites

  • An AWS account (free tier is sufficient)
  • Signed in to the AWS Management Console
  • Access to an email address for SNS notification confirmation

Step-by-Step Instructions

Important: Before creating a billing alarm, you must enable billing alerts. Go to Billing Dashboard → Billing Preferences (left sidebar) → check "Receive Billing Alerts" → Save. This only needs to be done once.
1
Navigate to SNS and create a topic.
In the AWS Console, use the search bar at the top and type SNS. Click on the Simple Notification Service to open it. Click Topics in the left sidebar, then click Create topic. Select Standard type and set the name to BillingAlerts. Click Create topic.
2
Create an email subscription to the topic.
On the topic detail page, click Create subscription. For Protocol, select Email. For Endpoint, enter your email address. Click Create subscription.
3
Confirm your email subscription.
Check your email inbox for a message from AWS Notifications. Click the Confirm subscription link in the email. You should see a confirmation page in your browser. Back in the SNS console, the subscription status should change to Confirmed.
4
Navigate to CloudWatch Alarms.
Use the search bar and type CloudWatch. Click on the CloudWatch service to open it. In the left sidebar, click Alarms → All alarms, then click Create alarm.
5
Select the billing metric.
Click Select metric. Navigate to Billing → Total Estimated Charge → USD. Select the checkbox next to the USD metric, then click Select metric.
6
Set the alarm threshold.
Under Conditions, select Static threshold type. Choose Greater than and enter 5 as the threshold value (or adjust to your comfort level). Click Next.
7
Configure the notification.
Under Notification, select In alarm as the alarm state trigger. Choose Select an existing SNS topic and select the BillingAlerts topic you created earlier. Click Next.
8
Name and create the alarm.
Set the alarm name to BillingThresholdAlarm. Optionally add a description like "Alert when estimated charges exceed $5." Review the configuration and click Create alarm.
9
Navigate to CloudWatch Dashboards.
In the CloudWatch left sidebar, click Dashboards. Click Create dashboard. Enter the name MyBillingDashboard and click Create dashboard.
10
Add a Number widget for billing.
In the Add widget dialog, select Number. Click Next, then choose Metrics as the data source. Navigate to Billing → Total Estimated Charge → USD. Select the metric and click Create widget.
11
Add a Text widget.
Click Add widget (the + button at the top). Select Text. In the markdown editor, add a note such as:
## Billing Monitor
This dashboard tracks my AWS spending. Alarm triggers at $5.
Click Create widget.
12
Save the dashboard.
Click Save at the top of the dashboard page. Your CloudWatch dashboard is now live with a billing number widget and an informational text widget.
Tip: The billing metric may take a few hours to populate if your account is brand new. The Number widget may initially show "No data" — this is normal. It will update as AWS processes your billing data.

Take a screenshot of the CloudWatch Dashboard page showing:

  • The billing Number widget displaying the estimated charge (or "No data" if new account)
  • The Text widget with your markdown note visible
  • The URL bar showing you are on the CloudWatch Dashboards page

Important: Your screenshot must include your AWS account name/ID visible in the top right corner of the AWS Console to verify completion.

Cleanup

🗑 Cleanup Steps:

Optional — we recommend keeping the billing alarm and dashboard as they are useful for ongoing cost monitoring. If you do want to remove them:

  1. Delete the dashboard: CloudWatch → Dashboards → select MyBillingDashboard → Delete
  2. Delete the alarm: CloudWatch → Alarms → select BillingThresholdAlarm → Delete
  3. Delete the SNS subscription: SNS → Subscriptions → select your email subscription → Delete
  4. Delete the SNS topic: SNS → Topics → select BillingAlerts → Delete

Key Takeaways

  • CloudWatch dashboards provide at-a-glance operational visibility — a core Operational Excellence practice
  • Billing alarms enable proactive response to unexpected costs before they grow
  • SNS topics decouple notification delivery from the alarm itself, following good architecture patterns
  • Combining monitoring (CloudWatch) with alerting (SNS) implements the "Respond to events" principle
