Lab 11: Configuring S3 Lifecycle Policies for Efficient Data Management

⏱ ~15 minutes ✅ Free Tier 📚 AWS Console Only

Overview

Sustainability in the cloud means reducing unnecessary resource consumption. One of the most impactful practices is managing data lifecycle — automatically moving data to cheaper, more energy-efficient storage classes as it ages, and deleting it when no longer needed. In this lab, you'll create an S3 bucket, explore storage classes, and configure a lifecycle policy that automates data transitions. This connects to the Sustainability pillar principles of Data Storage and Utilization & Scaling from the Week 2 lecture.

Objectives

  • Understand S3 storage classes and their energy/cost tradeoffs
  • Create a lifecycle rule that automates data transitions across storage tiers
  • Configure automatic expiration of old data to reduce storage footprint
  • Connect data lifecycle management to sustainability and environmental impact

Prerequisites

  • An AWS account (free tier is sufficient)
  • Signed in to the AWS Management Console

Step-by-Step Instructions

1
Navigate to S3 and create a bucket.
In the AWS Console, search for S3 and open the service. Click Create bucket. For the bucket name, enter yourname-sustainability-lab-2026 (replace "yourname" with your actual name). Leave all other settings as defaults and click Create bucket.
2
Upload test files to the bucket.
Click into your new bucket and click Upload. Add 3–4 small test files (text files, small images, etc. — anything under 1MB total). Click Upload to complete.
3
Explore storage classes.
Click on one of your uploaded files and scroll down to the Properties section. Note that the Storage class is Standard by default. Understanding storage classes is key:
  • S3 Standard — Frequently accessed data, highest availability, highest cost
  • S3 Standard-IA (Infrequent Access) — Lower cost for data accessed less often, retrieval fees apply
  • S3 Glacier Instant Retrieval — Archive data that needs millisecond access, much cheaper
  • S3 Glacier Flexible Retrieval — Archive data accessed 1–2 times per year, minutes-to-hours retrieval
  • S3 Glacier Deep Archive — Lowest cost, data accessed once or twice a year, 12–48 hour retrieval
4
Go to the Management tab.
Navigate back to your bucket and click on the Management tab. This is where you configure lifecycle rules.
5
Create a lifecycle rule.
Click Create lifecycle rule.
6
Name the rule.
Enter the lifecycle rule name: optimize-storage-costs.
7
Set the rule scope.
Choose Apply to all objects in the bucket. Check the acknowledgement checkbox to confirm you want this rule to apply to all objects.
8
Select lifecycle rule actions.
Under Lifecycle rule actions, check the following:
  • Transition current versions of objects between storage classes
  • Expire current versions of objects
9
Configure transitions.
Under the transitions section, add the following:
  • Add transition: Standard-IA after 30 days
  • Add transition: Glacier Flexible Retrieval after 90 days
10
Configure expiration.
Set Expire current versions of objects after 365 days. Also check Delete expired object delete markers.
11
Create the rule and review.
Click Create rule. Review the rule summary on the Management tab to confirm the transitions and expiration are configured correctly.
Note: These transitions won't happen immediately — they take effect after the specified number of days. For this lab, we're configuring the policy to see how it works. In a real environment, data would automatically move through the tiers: Standard → Standard-IA (30 days) → Glacier (90 days) → Deleted (365 days).
Sustainability Impact: How does this help sustainability? Each storage tier uses progressively less energy for storage. Glacier stores data on high-density magnetic media in optimized facilities. By automatically tiering data, you reduce the energy footprint of your storage. And by expiring data you no longer need, you eliminate storage consumption entirely — the greenest byte is the one you don't store.

Cleanup

🗑 Cleanup Steps:
  1. Go to the bucket's Management tab → select the lifecycle rule → Delete
  2. Go to the bucket → select all objects → Delete → confirm
  3. Go back to S3 → select the bucket → Delete → confirm by typing the bucket name

Key Takeaways

  • Lifecycle policies automate data management — reducing manual effort and ensuring data moves to efficient storage tiers
  • Each S3 storage class represents a tradeoff between access speed, cost, and energy consumption
  • Automatic expiration prevents data hoarding — deleting data you no longer need is the most sustainable action
  • Sustainability in the cloud is about using only the resources you need, for only as long as you need them
