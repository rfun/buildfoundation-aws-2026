Lab 7: Exploring EC2 Instance Types and Launching an Instance

⏱ ~25 minutes ✅ Free Tier 📚 AWS Console Only

Overview

Choosing the right EC2 instance type is a core Performance Efficiency decision. Different instance families are optimized for different workloads — general purpose, compute-optimized, memory-optimized, and more. In this lab, you'll explore instance type families, launch a free-tier eligible instance, and examine its monitoring metrics. This connects to the Performance Efficiency pillar principles of "Architecture selection" and "Compute & Hardware" from the Week 2 lecture.

Objectives

  • Understand EC2 instance type families and their use cases
  • Launch a free-tier eligible EC2 instance
  • Explore CloudWatch monitoring metrics for the instance
  • Understand how right-sizing instances optimizes performance efficiency

Prerequisites

  • An AWS account (free tier is sufficient)
  • Signed in to the AWS Management Console
  • Note: This lab launches an EC2 instance — remember to terminate it at the end to stay within free tier

Step-by-Step Instructions

1
Navigate to EC2 in the AWS Console.
Use the search bar at the top and type EC2. Click on the EC2 service to open it. In the left sidebar, click Instance Types.
2
Explore different instance families.
Browse the instance types list by filtering or scrolling. Key families to understand:
  • t3/t2 (General Purpose, Burstable) — good for web servers, small databases, dev environments
  • m5/m6i (General Purpose, Fixed) — balanced compute, memory, networking for application servers
  • c5/c6i (Compute Optimized) — best for batch processing, gaming servers, scientific modeling
  • r5/r6i (Memory Optimized) — ideal for in-memory databases, real-time big data analytics
3
Filter for free tier eligible instances.
Use the filter: add a filter for "Free tier eligible" = "Yes" to see which instance types qualify for the free tier.
4
Navigate to Launch Instance.
Go to EC2 → click "Launch Instance" (top right of the EC2 dashboard).
5
Name your instance.
Enter the name: performance-lab-instance
6
Select the AMI.
Under AMI, select "Amazon Linux 2023 AMI" (should be marked "Free tier eligible").
7
Choose the instance type.
Under Instance type, select t2.micro (Free tier eligible). If t2.micro is not available in your region, select t3.micro.
8
Configure key pair.
Under Key pair, select "Proceed without a key pair" (we won't be SSH-ing into this instance).
9
Configure network settings.
Keep the default VPC and ensure "Auto-assign public IP" is set to Enable. Click "Edit" on the security group → Create a new security group → allow SSH (port 22) from "My IP" only.
10
Launch the instance.
Leave all other settings as defaults → click "Launch instance".
11
Wait for the instance to start.
Click "View all instances" → wait for the instance state to change to "Running" (may take 1–2 minutes — refresh the page).
12
Review instance details.
Click on your instance ID to open the detail page. Review: Instance type, Availability Zone, Public IPv4 address, and Status checks (2/2 checks passed).
13
Explore monitoring metrics.
Click the "Monitoring" tab → explore the CloudWatch metrics: CPUUtilization, NetworkIn, NetworkOut, DiskReadOps, etc. (Metrics may take 5 minutes to populate.)
14
Understand monitoring intervals.
Note: these are basic monitoring metrics at 5-minute intervals. Detailed monitoring (1-minute intervals) is available for an additional cost.
⚠ Important: Remember to terminate this instance when you're done! EC2 instances running 24/7 will consume your 750 free tier hours. Terminating immediately after the lab keeps your usage minimal.

Completion Screenshot

📷 What to capture:

Take a screenshot of the EC2 instance detail page showing:

  • Instance state = Running
  • Instance type = t2.micro (or t3.micro)
  • The Monitoring tab with at least one metric graph visible

Important: Your screenshot must include your AWS account name/ID visible in the top right corner of the AWS Console to verify completion.

Cleanup

🗑 Cleanup Steps:
  1. Navigate to EC2 → Instances
  2. Select "performance-lab-instance" → Instance state → Terminate instance → confirm
  3. Wait for the instance to show "Terminated" state
  4. Navigate to EC2 → Security Groups → find the security group created for this lab (it will have "launch-wizard" in its name or the name you chose) → Actions → Delete security group → confirm
⚠ Important: Always verify your instance reaches "Terminated" state. A "Stopped" instance still reserves resources. Only "Terminated" fully releases all resources.

Key Takeaways

  • EC2 instance families are purpose-built for different workloads — choosing the right one is key to performance efficiency
  • The t2/t3 burstable instances are cost-effective for variable workloads but may throttle under sustained load
  • CloudWatch metrics help you monitor whether your instance is right-sized (under-utilized = wasting money, over-utilized = poor performance)
  • Right-sizing is an ongoing process: start small, monitor, and adjust based on actual usage patterns
