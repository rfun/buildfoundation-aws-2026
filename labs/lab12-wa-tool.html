Lab 12: Running an AWS Well-Architected Framework Review

⏱ ~25 minutes ✅ Free Tier 📚 AWS Console Only

Overview

The AWS Well-Architected Tool brings together everything from the Week 2 lecture into a hands-on review process. It provides a structured way to evaluate workloads against the six pillars using guided questions and generates improvement plans. In this lab, you'll define a sample workload, answer sustainability-focused questions, and explore the improvement plan. This connects to the Well-Architected Tool concepts from Week 2 (Slide 25) and the continuous improvement cycle: Identify → Evaluate → Prioritize → Test → Deploy → Measure.

Objectives

  • Define a workload in the AWS Well-Architected Tool
  • Answer framework review questions for the Sustainability pillar
  • Interpret the risk assessment and improvement plan generated by the tool
  • Understand how the tool supports the continuous review cycle discussed in Week 2

Prerequisites

  • An AWS account (free tier is sufficient)
  • Signed in to the AWS Management Console
  • Familiarity with the 6 WAF pillars from the Week 2 lecture

Step-by-Step Instructions

1
Navigate to the AWS Well-Architected Tool.
In the AWS Console, use the search bar at the top and type Well-Architected. Click on the AWS Well-Architected Tool service to open it.
2
Define a new workload.
Click "Define a workload" and fill in the following details:
  • Workload name: Sample Web Application
  • Description: A simple web application for learning the Well-Architected review process
  • Review owner: your name or email
  • Environment: Pre-production
  • Regions: select your current region

Click "Next" → on the Lenses page, ensure "AWS Well-Architected Framework" is checked → click "Define workload".

3
Open the workload and start the review.
Click on "Sample Web Application" to open it → click "Continue reviewing" → select the "AWS Well-Architected Framework" lens.
4
Navigate to the Sustainability pillar.
In the left sidebar of the review, find and click on the Sustainability pillar section to begin answering sustainability-focused questions.
5
Answer the sustainability questions.
Work through the sustainability questions using the following guidance for a sample web application:
  • SUS 1 (Region selection for sustainability): Select "None of these" if you haven't specifically chosen a region for sustainability reasons. In practice, you would consider AWS regions powered by renewable energy.
  • SUS 2 (User behavior patterns): Select "None of these" if just starting out. For a real app, you'd consider scaling down during low-traffic periods.
  • SUS 3 (Software and architecture patterns): Select "None of these" for a simple app. Advanced practices include using async processing, efficient algorithms, and right-sized resources.
6
Continue through additional sustainability questions.
Continue through any additional sustainability questions, selecting the best answers for a simple pre-production web app. It's OK to select "None of these" for questions where you haven't implemented the best practice yet — the tool will flag these as improvement opportunities.
7
Save and exit the review.
After answering the sustainability questions, click "Save and exit" to save your progress.
8
Review the workload dashboard.
Go to the workload Overview/Dashboard and review the pillar summary showing risk counts: High Risk (red), Medium Risk (yellow), and No Risk (green) for each pillar. The Sustainability pillar should show some risks based on your answers.
9
Explore the improvement plan.
Click on the "Improvement plan" tab and review the suggested improvements for Sustainability. Each recommendation includes a description and a link to AWS documentation with detailed guidance.
10
Briefly explore other pillars.
Navigate to 1–2 other pillars (e.g., Security, Reliability) and note how the questions are structured: each question maps to a best practice, and your answers determine your risk level.
Note: Don't worry about having "High Risk" results — this is expected for a sample workload! The purpose of the tool is to identify areas for improvement. In a real environment, you would use the improvement plan to prioritize and address risks over time, following the Identify → Evaluate → Prioritize → Test → Deploy → Measure cycle from the Week 2 lecture.
Note: The Well-Architected Tool is free and doesn't create or modify any AWS resources. It's purely an assessment and planning tool that generates recommendations based on your answers.

  • The Sustainability pillar showing at least some risk items based on the review answers

Cleanup

🗑 Cleanup Steps:
  1. Navigate to the Well-Architected Tool → Workloads
  2. Select "Sample Web Application"
  3. Click "Delete" → confirm by typing the workload name → Delete

Key Takeaways

  • The Well-Architected Tool provides a structured, repeatable process for evaluating workloads against all 6 pillars
  • Risk ratings (High, Medium, None) help you prioritize which improvements to tackle first
  • The generated improvement plan includes specific, actionable recommendations with documentation links
  • Well-Architected reviews should be done regularly — they're part of the continuous improvement cycle, not a one-time exercise
