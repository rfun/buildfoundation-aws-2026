AWS Cloud Security – Recent Breaches

2019 – Capital One

Misconfigured Web Application Firewall that allowed the attacker to grab AWS instance metadata including temporary security credentials assigned to the IAM role. Allowed the attacker to access sensitive data in S3 buckets including 100 Million customer records, SSN, emails etc. Cost: $150 million

2022 – Pegasus Data Breach

Misconfigured S3 bucket which exposed 6.5 TB of sensitive data including crew members' personal information and operational details.

2021 – Twitch

Over 125 GB of data was leaked due to a server configuration error which allowed public access to internal systems.

2025 – Codefinger Ransomware

Compromised AWS credentials allowed attackers to use a custom AWS key to encrypt data in users' S3 buckets. Since a custom key was used, it made it impossible for the victims to decrypt their data without the attackers' keys.

AWS Cloud Security – Common Problems and Suggestions

Encryption

  • RDS is by default created without encryption
  • EBS volumes are not encrypted by default
  • S3 should employ at rest encryption
  • Use KMS managed keys for encryption

Connecting EC2 directly to the internet

  • Create EC2 instances in a private Subnet, protected by Security groups and ACL rules
  • If internet access is needed (Nginx), use a WAF and load balancer

S3 Buckets – Public Access

  • S3 configuration errors or incomplete security settings can allow public access to buckets
  • Use pre-signed URLs for uploads/downloads, block all public access
  • Regularly audit permissions

Leaving Insecure ports open

SSH, HTTP, FTP, TelNet ports should generally be shut. Only allow SSH access from trusted IPs.

No MFA for IAM Users

Ensure strict password policies and MFA standards on any user that has access to the AWS CLI/UI.

AWS Cloud Security – Amazon Inspector

What does it do?

Discovers Amazon EC2 instances, containers, and Lambda functions, and scans them for software vulnerabilities and unintended network exposure.

Pricing

EC2 (per instance/per month): SSM – 1.2528, Agentless – 1.7496
ECR: $0.09 per image, $0.01 per rescan
Lambda: Standard (only package scans): 0.3$ per lambda/month, Code scanning: 0.9$ per lambda/month

Features

  • Checks installed software on EC2 instances and container images, packages in Lambda code and compares it with over 50 CVE (Common Vulnerabilities & Exposures). Re-runs the scan if a new CVE is published
  • Can use AWS System Manager agent (SSM agent) or perform agentless scanning by taking EBS snapshots
  • Can prioritize and report vulnerabilities to AWS Security Hub and AWS EventBridge
  • Detects if a reported vulnerability has been patched/remediated and closes the finding automatically
  • Can provide generative AI powered remediation/code patches for Lambda functions

High Availability (HA) – AZ/Regions
Availability Zone – Recap
  • Built for high reliability so that nothing short of a natural disaster brings one down
  • Located close to one another and connected via low-latency high-bandwidth links to ensure low latency between the facilities
  • Located far enough from each other that a single natural disaster cannot bring all of them down at the same time

Reliability

Resiliency is the ability of a workload to recover from infrastructure or service disruptions, dynamically acquire computing resources to meet demand, and mitigate disruptions, such as misconfigurations or transient network issues.

Reliability – Backup Options
  • Manual Backups – EBS Snapshot, RDS etc.
  • Automated Backups using Amazon Data Lifecycle Manager for EBS
  • Automated snapshots using RDS backup policy – Generally limited to 30 days
  • Point in time restore for RDS – Can have up to 5 minutes of data loss
  • AWS Backup
